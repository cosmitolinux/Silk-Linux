Silk Linux is an Arch-based GNU/Linux distribution with an obsessive focus on driver reliability and long-term system stability. Built on pacman and the Arch ecosystem — with pre-built binary packages — every kernel update is tested against our hardware compatibility matrix before reaching your machine. No compiling. Just a rock-solid system.

Driver Reliability

Every driver in the Silk tree is individually reviewed, tested against five kernel configurations, and signed before packaging. No experimental drm-next blobs ship without a stability gate.

Kernel Stability

Silk ships a hardened, long-lived kernel branch. Patches are cherry-picked from stable/, regression-tested in CI, and released on a predictable 6-week cadence. No surprises.

Hardware First

Before any package lands in stable, it is validated on our 200-node hardware test farm spanning server, desktop, embedded, and ARM targets. Real hardware, real drivers.

Arch-Based

Built on the Arch Linux ecosystem with pacman as the package manager. You get the speed and simplicity of binary packages, access to the AUR, and Arch's rolling-release model — without the complexity of compiling from source.

Quick Installation

Boot the Silk ISO, run the guided installer, and be up in minutes. Packages are pre-built binaries — no compiling required. Driver detection runs automatically at install time.

# As soon as you boot up, run:
silk install system

# Once open, go through it and finish it

# Now if you wanted to install manually with drivers,
# do it like Arch — but use silkdvrfetch instead:
silkdvrfetch -ETR /path/to/drivers >> /mnt

What Sets Silk Apart

  • Arch-based with pacman — pre-built binary packages, no compilation required
  • Full AUR compatibility via yay and paru — access the entire Arch ecosystem
  • Dedicated silk-drivers pacman repository with individually CI-tested driver packages
  • Kernel built with CONFIG_LOCKDEP, CONFIG_KASAN, and CONFIG_KMSAN validation passes before release
  • Automated regression bisection — broken drivers are caught before they reach stable
  • Hardware compatibility matrix updated daily from the 200-node test farm
  • Real-time kernel panic reports aggregated and triaged by the Silk kernel team
  • Binary kernel modules are cryptographically signed; unsigned modules are rejected at boot
  • Out-of-tree driver submissions go through a structured review and testing process
  • Rolling release model following Arch — always up to date, never a version upgrade
  • LTS kernel branch with a 6-year support commitment and zero ABI breaks within a major

Silk Packages — Universal Tools, Any Linux

Silk ships a suite of first-party command-line tools — collectively called silk packages — that run on any Linux distribution, not just Silk. They are thin, dependency-light wrappers that unify common tasks under a consistent, memorable interface. Install once, use everywhere.

Core Commands
sk g <package>
Get / Install a package
Resolves the correct package manager for your distro (pacman, apt, dnf, zypper…) and installs the package. One command, any Linux.
sk g firefox  # installs firefox on any distro
sk g neovim git curl
sk r <package>
Remove a package
Removes the package and optionally its orphaned dependencies. Asks for confirmation before touching anything.
sk r firefox  # remove and clean up
sk r --purge oldpkg  # purge config files too
sku
Update the system
Syncs repos and upgrades all packages in one shot. On Silk, also checks the driver compatibility matrix before applying kernel updates.
sku  # full system update
sku --dry-run  # preview what would change
skgit <url>
Reliable Git clone
A smarter git clone — retries on failure, verifies repo integrity, supports shallow/sparse clones, and optionally validates GPG commit signatures.
skgit https://github.com/user/repo
skgit --verify-gpg --shallow https://github.com/user/repo
sk(output) notation
Every sk command produces structured, color-coded output. Prefixes tell you exactly what happened at a glance: [OK] success [ERR] failure with reason [WARN] non-fatal warning [INFO] status message
